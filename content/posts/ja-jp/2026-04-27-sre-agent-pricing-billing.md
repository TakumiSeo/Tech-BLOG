# はじめに

MS Learn の [「Azure SRE エージェントの価格と課金」](https://learn.microsoft.com/ja-jp/azure/sre-agent/pricing-billing) は機械翻訳の影響もあり読みづらいため、課金構造を draw.io 図と具体例の計算で整理する。本記事の数値・レート・タスク区分は同ドキュメント (Last updated 2026-04-23、AAU レートは 2026-04-15 時点) の記載のみを根拠としており、推論を含めない[^1]。

# 課金の基本構造

SRE Agent の請求は、すべて **AAU (Azure Agent Unit)** という統一単位で計上される。AAU は事前ビルド済み Azure エージェント全般で共通の指標で、月額請求は次の 2 種類の合算となる[^1]。

| 区分 | 種類 | 課金条件 |
| ---- | ---- | -------- |
| ① Always-on フロー | 固定コスト | エージェント作成 → 削除までエージェント存在時間に対して課金 |
| ② Active フロー   | 変動コスト | エージェントが実際に処理 (推論・ツール呼び出し等) を行ったトークン量に対して課金 |

USD 換算は Azure 料金ページで **1 AAU = $0.10** と明示されている (Always-on は 4 AAU/h × $0.10 = $0.40/h)[^2]。リージョン・契約形態・為替により実際の請求額は変動するため、正確な見積もりは Azure 料金計算ツールで確認する必要がある。

![SRE Agent 課金モデル](images/sre-agent-pricing-billing/billing-overview.drawio.svg)

# ① Always-on フロー (固定コスト)

- レート: **エージェント 1 つあたり 4 AAU / 時間**[^1]
- エージェントを作成した時点から削除するまで継続して課金される。
- 「エージェントが常に作業をしている」という意味ではなく、プロビジョニング状態を維持するためのベースラインコストとして扱われる。
- **重要:** エージェントを「停止 (Stop)」しても Always-on 課金は継続する。完全に止めるには「削除 (Delete)」が必要[^1]。

ドキュメント記載のレートから、時間スケール別の固定コストを単純計算すると以下になる (1 AAU = $0.10[^2] で USD 換算)。

| 期間 | 計算 | AAU | USD |
| ---- | ---- | --- | --- |
| 1 時間 | 4 × 1     | 4 AAU | $0.40 |
| 1 日 (24h) | 4 × 24 | 96 AAU | $9.60 |
| 30 日換算 | 4 × 24 × 30 | 2,880 AAU | $288.00 |
| 31 日換算 | 4 × 24 × 31 | 2,976 AAU | $297.60 |

# ② Active フロー (変動コスト)

エージェントが処理を行うたびに、LLM トークンの消費量に応じて Active フロー AAU が消費される。チャットからの対話、自動化、非同期バックグラウンド処理のいずれであっても、**エージェントが実際に処理に費やした時間のみ**が対象となる。ユーザーの応答待ちや承認待ちの時間は課金されない[^1]。Active フローのカウンタは **カレンダー月初にリセット** される[^1]。

## 4 つのトークン種別

トークンは 4 種類に分けて個別に計測される[^1]。

| 種別 | 内容 |
| ---- | ---- |
| Input | モデルに送信されるトークン (プロンプト・ツール結果・コンテキスト) |
| Output | モデルが生成するトークン (応答・推論) |
| Cache read | プロンプトキャッシュから供給されたトークン (繰り返しコンテキスト) |
| Cache write | 将来の再利用のためにキャッシュへ書き込まれたトークン |

タスク 1 件分の Active フロー AAU は、上記 4 種別の AAU 合計値となる。

## モデル別 AAU レート (1M トークンあたり)

2026-04-15 時点で MS Learn に記載されているレートは次のとおり[^1]。

| モデル | Input | Output | Cache read | Cache write |
| ------ | ----- | ------ | ---------- | ----------- |
| Claude Opus 4.6 | 100 AAU | 500 AAU | 10 AAU | 125 AAU |
| GPT 5.3 Codex   | 35 AAU  | 280 AAU | 3.5 AAU | — |
| GPT 5.2         | 35 AAU  | 280 AAU | 3.5 AAU | — |

> モデルプロバイダー (Anthropic / OpenAI など) はエージェント単位で設定し、対応モデルが AAU レートを決定する[^1]。GPT 系の Cache write 列は MS Learn 上「—」表記で、本記事ではそのまま転記する (Cache write が課金対象か否かは推論しない)。

## 計算式

トークン種別ごとに以下を算出し、4 種別を合算したものが当該タスクの Active フロー AAU となる。

$$
\text{AAU}_{\text{種別}} = \frac{\text{使用トークン数}}{1{,}000{,}000} \times \text{AAU レート}_{\text{種別}}
$$

# 具体例: 「Quick question」を 1 回実行 (モデル別比較)

MS Learn の「タスクの種類別のアクティブ フロー」表に記載された Quick question 行 (Input ~20K / Output ~2K / Cache read ~15K / Cache write ~5K) を使って計算する[^1]。USD は **1 AAU = $0.10**[^2] で換算。

## (A) Claude Opus 4.6 の場合

| 種別 | トークン数 | レート / 1M | 計算 | AAU | USD |
| ---- | ---------- | ----------- | ---- | --- | --- |
| Input       | 20,000 | 100 | 20,000 / 1,000,000 × 100 | 2.000 | $0.2000 |
| Output      |  2,000 | 500 |  2,000 / 1,000,000 × 500 | 1.000 | $0.1000 |
| Cache read  | 15,000 |  10 | 15,000 / 1,000,000 × 10  | 0.150 | $0.0150 |
| Cache write |  5,000 | 125 |  5,000 / 1,000,000 × 125 | 0.625 | $0.0625 |
| **合計** |  |  |  | **3.775 AAU** | **$0.3775** |

MS Learn の「数学のしくみ」表でも同じ 3.775 AAU が示されており、本文では「約 3.8 AAU」と表記されている[^1]。

## (B) GPT 5.3 Codex の場合

GPT 系モデルは MS Learn の AAU レート表で **Cache write 列が「—」表記** となっている[^1]。本記事では Cache write を算入しない方針で計算する (推論を加えない)。

| 種別 | トークン数 | レート / 1M | 計算 | AAU | USD |
| ---- | ---------- | ----------- | ---- | --- | --- |
| Input       | 20,000 | 35  | 20,000 / 1,000,000 × 35  | 0.7000 | $0.0700 |
| Output      |  2,000 | 280 |  2,000 / 1,000,000 × 280 | 0.5600 | $0.0560 |
| Cache read  | 15,000 | 3.5 | 15,000 / 1,000,000 × 3.5 | 0.0525 | $0.0053 |
| Cache write |  5,000 |  —  | 算入せず | — | — |
| **合計** |  |  |  | **1.3125 AAU** | **$0.13125** |

MS Learn の Quick question 行は GPT を「~1.6 AAU」と概数で示しており[^1]、本計算 (1.3125 AAU) との差異は概算丸めおよび Cache write の扱いに起因する可能性があるが、ドキュメント上明示されていないため断定しない。

## (A) と (B) の比較

| 区分 | Opus 4.6 | GPT 5.3 Codex | 比 (GPT / Opus) |
| ---- | -------- | ------------- | ---------------- |
| Active AAU (1 回) | 3.775 | 1.3125 | 約 34.8% |
| Active USD (1 回) | $0.3775 | $0.13125 | 約 34.8% |

## 1 日の合計コストイメージ

エージェントを 1 日 (24 時間) 稼働させ、上記 Quick question を 1 回実行した場合の合計を単純計算すると以下になる。

| 区分 | Opus 4.6 | GPT 5.3 Codex |
| ---- | -------- | ------------- |
| Always-on (24h × 4 AAU/h) | 96.000 AAU / $9.60 | 96.000 AAU / $9.60 |
| Active (Quick question 1 回) | 3.775 AAU / $0.3775 | 1.3125 AAU / $0.13125 |
| **合計 / 日** | **99.775 AAU / $9.9775** | **97.3125 AAU / $9.73125** |

1 日 1 回の Quick question 程度であれば、固定コストの Always-on が支配的でモデル選択による差は限定的である点が確認できる。

# タスクの種類別の Active フロー (MS Learn 例)

参考として、MS Learn が示す代表的タスクの AAU レンジは次のとおり[^1]。Claude Opus 4.6 / GPT (5.2 / 5.3 Codex 共通) の概算値が併記されている。

| タスク | Input | Output | Cache read | Cache write | Opus 4.6 | GPT | 例 |
| ---- | ----- | ------ | ---------- | ----------- | -------- | --- | ---- |
| Quick question | ~20K | ~2K | ~15K | ~5K | ~3.8 | ~1.6 | "最近のアラートを表示する" |
| Incident investigation | ~200K | ~15K | ~150K | ~50K | ~35.5 | ~13.7 | Azure Monitor 起点の自動インシデント |
| Full remediation | ~500K | ~40K | ~400K | ~100K | ~86.5 | ~33.9 | "失敗したデプロイを診断して修正する" |

Claude Opus 4.6 で各レートを当てはめると、Full remediation は次のとおり厳密に 86.5 AAU と一致する。

- Input: 500,000 / 1,000,000 × 100 = **50.0**
- Output: 40,000 / 1,000,000 × 500 = **20.0**
- Cache read: 400,000 / 1,000,000 × 10 = **4.0**
- Cache write: 100,000 / 1,000,000 × 125 = **12.5**
- 合計: **86.5 AAU** ✅

# コスト管理の仕組み

## ポータルでの可視化

Settings > Agent consumption 画面で次が確認できる[^1]。

- Monthly AAU limit (Always-on と Active の合計割当)
- Total active flow consumption (進捗バー)
- Daily active flow consumption (日次の棒グラフ)
- Token usage breakdown (Input / Output / Cache read / Cache write 別の合計)

## Active フローの上限設定

「Change AAU allocation」から **Active フローのみ** に対して月次の AAU 上限 (最大 100 万 AAU) を設定できる[^1]。

- 上限到達時、エージェントはチャット・アクションが翌月まで使用不可となる。
- 上限到達中も Always-on 課金は継続する。
- 割り当ては随時変更可能。

## アクション別の課金影響

| アクション | Active フロー | Always-on フロー | 備考 |
| ---------- | ------------- | ---------------- | ---- |
| 予算上限到達 | 停止 | 継続課金 | 月初に自動リセット |
| エージェント停止 (Stop) | 停止 | 継続課金 | Settings > Basics で再開可 |
| エージェント削除 (Delete) | 停止 | 停止 | 新規エージェント作成が必要 |

複数エージェント・リソース横断の詳細な内訳は Microsoft Cost Management で確認する[^1]。

# コスト最適化のヒント (MS Learn 抜粋)

ドキュメントに明記されている最適化アクションのみを列挙する[^1]。

- **コンテキストを追加する**: スキル・ナレッジ・ドキュメントを与えてトークンの無駄を削減。永続メモリで時間経過とともに効率化。
- **応答プランで対応インシデントを絞る**: 重要度・サービス・キーワードで Azure Monitor アラートをフィルタし、一致したインシデントのみ調査させる。
- **スケジュールタスクをバッチ化する**: ポーリングではなく日次・週次に。
- **自動化前にチャットで試す**: 設定不備の自動化が繰り返し走ると AAU を浪費する。
- **使っていないエージェントは Stop**: Active フロー消費を停止 (ただし Always-on は継続)。
- **不要エージェントは Delete**: すべての課金が即時停止。

## モデル選択について (MS Learn の見解)

- Claude Opus 4.6: AAU レートは高いが、複雑なインシデント調査・根本原因分析でツール呼び出しが少なく結論に至るため、トークン単価の高さを相殺できることが多い[^1]。
- GPT 系: スケジュール化されたコンプライアンスチェックなど、深さよりコスト効率が重要なシンプル・高頻度タスクに向く[^1]。

# まとめ

- 課金単位は **AAU** に統一されており、**1 AAU = $0.10**[^2]。月額は **Always-on (4 AAU/h × 存在時間 = $0.40/h) + Active (4 種別トークン × モデル別レート × $0.10)** の 2 階建て構造。
- Always-on は **Stop しても止まらない**。完全停止には Delete が必要。
- Active フローはユーザー応答待ち時間を含まず、月初にカウンタリセット、上限は最大 100 万 AAU まで設定可能。
- MS Learn の例値ベースで計算すると、同一ワークロードに対する Opus / GPT の AAU 比は Quick question 3.8 / 1.6 = **2.375 倍**、Incident investigation 35.5 / 13.7 ≒ **2.591 倍**、Full remediation 86.5 / 33.9 ≒ **2.552 倍**[^1]。
- 円ドル建ての価格・リージョン別価格は MS Learn には記載されておらず、Azure 料金計算ツールでの確認が必須[^1]。

[^1]: Azure SRE エージェントの価格と課金 (Last updated 2026-04-23、AAU レート 2026-04-15 時点), https://learn.microsoft.com/ja-jp/azure/sre-agent/pricing-billing

[^2]: Azure SRE Agent pricing (Always-on flow: 4 AAU × $0.10 = $0.40 per hour per agent / Active flow: $0.10 per AAU), https://azure.microsoft.com/en-us/pricing/details/sre-agent/
